## Rule
`LeatherBook` in `components/BookshelfLibrary.tsx` layers a single reusable photographed leather-grain texture (`assets/images/leather-texture.jpg`) as an `Image` (`resizeMode="cover"`) under a per-category `LinearGradient` tint, foil-stamped gold ornaments (double hairline frame + SVG corner flourishes + `OrnateDivider`), and a large embossed title as the cover's hero element (small call-number/roman-numeral badge, if any, must not collide with the top eyebrow text — give it its own row or drop it). The shelf itself uses two separate generated wood photos, not gradients: a vertical-grain `wood-wall-texture.jpg` behind the books (cabinet/back wall) and a horizontal-grain `wood-shelf-texture.jpg` under each row (`ShelfPlank`), each with a thin dark tint overlay for depth.

**Why:** An earlier version tinted the leather texture at ~90-95% opacity and added large flat translucent "spotlight" discs plus an oversized numeral as the cover's centerpiece, with a flat gradient standing in for the wood shelf. The user rejected that result as "glossy/glass-morphism, not real leather" against a reference photo — the high tint opacity washed out the grain, and the flat vector shelf broke the illusion regardless of the leather quality. Real photographed wood (generated once per role, same reuse pattern as the leather texture) plus lower tint opacity (roughly 78-92%, base-to-deep) plus vector foil ornaments instead of glass discs is what reads as photoreal.

**How to apply:** If more categories/books are added, reuse the existing `leather-texture.jpg`/wood textures rather than generating new ones per item — the tint color and title text are what differentiate books, not the underlying grain. If the shelf/cover still reads as flat after a redesign, check tint opacity first and check for any large soft-edged translucent shape (glow discs, giant numerals) competing with the texture — those are the recurring "fake" tell, more so than the texture asset itself.

It still reflects the pre-redesign flat-gradient-shelf/high-opacity-tint version — if web parity is requested, port the wood-photo shelf, lower tint opacity, and SVG foil ornaments the same way this file already ports `CATEGORY_INFO`/roman numerals/flavor text (taglines stay English on web, Portuguese on mobile).
