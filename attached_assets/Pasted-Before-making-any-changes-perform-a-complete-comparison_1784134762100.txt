Before making any changes, perform a complete comparison between the current Website and the Mobile App.

Do NOT redesign the application.

Do NOT change colors.

Do NOT change typography.

Do NOT change spacing.

Do NOT change the navigation.

Do NOT change the visual identity.

The current mobile application is the primary product.

The website is NOT the product.

The website is only a marketing and informational experience.

The mobile app is where users study, read, listen and learn.

==================================================
PROJECT STRATEGY
==================================================

Bread&Light will become a Mobile-First platform.

The website should only serve as:

• Landing Page
• App presentation
• Features overview
• Screenshots
• Download links
• Privacy Policy
• Terms of Service
• Support
• Contact

The website should NOT contain exclusive learning features.

All learning experiences belong inside the mobile application.

==================================================
AUDIT
==================================================

Inspect both projects.

Compare:

Website

vs

Mobile App

Identify every feature that exists on the website but is missing inside the mobile application.

Create a report listing:

• Existing only on Website
• Existing only on Mobile
• Duplicated
• Missing on Mobile

==================================================
MIGRATION
==================================================

Move every useful learning feature from the Website into the Mobile App.

Do NOT remove anything already working inside the app.

Only ADD missing features.

Preserve the current mobile design.

Preserve the current navigation.

Preserve every color.

Preserve typography.

Preserve spacing.

Preserve all interactions.

==================================================

• Content carousels

==================================================
IMPLEMENTATION
==================================================

First produce the comparison report.

Then migrate only the missing features.

Reuse as much code as possible.

Avoid duplication.

Maintain performance.

Maintain responsiveness.

Do not create parallel implementations.

The Mobile App becomes the primary experience.

The Website becomes the presentation layer.
